# <a name="review-data-for-a-subject-rights-request"></a>주체 권한 요청에 대한 데이터 검토

Microsoft Priva에서 주체 권한 요청([자세한 정보](subject-rights-requests-create.md))을 만든 후 주체 권한 요청 솔루션은 입력을 사용하여 조직의 Microsoft 365 환경에서 데이터 주체에 대한 일치 항목을 찾습니다. 이 데이터가 컴파일되면 결과를 검토하고, 포함할 항목을 선택하고, 필요에 따라 정보를 수정할 수 있습니다. 이러한 단계는 Priva 인터페이스를 통해 여러 사용자가 공동 작업할 수 있습니다.

## <a name="step-1-review-request-details-and-monitor-progress"></a>1단계: 요청 세부 정보 검토 및 진행률 모니터링

검색의 초기 결과를 보려면 [Microsoft Purview 규정 준수 포털](https://compliance.microsoft.com/) 의 Priva 영역으로 이동하여 **주체 권한 요청을** 엽니다. 열려 있는 모든 주체 권한 요청 목록은 이 기본 페이지에서 찾을 수 있습니다.

목록에서 요청을 선택하여 요청 세부 정보를 확인합니다. 여기서 요청의 속성, 검색 결과 및 요청 상태에 대해 자세히 알아볼 수 있습니다. 이 페이지는 찾은 파일을 관리하고, 보고서 및 내보내기를 만들고, 요청을 완료하는 작업을 수행하고 공동 작업하는 허브가 됩니다.

요청 세부 정보 페이지의 **개요** 탭에는 요청에 대한 세부 정보, 현재 단계를 보여 주는 진행률 표시기 및 찾은 데이터에 대한 주요 정보가 제공됩니다. 이 페이지의 타일에는 다음이 포함됩니다.

##### <a name="details"></a>세부 정보

**세부 정보** 카드는 마감일, 생성 날짜, 설명 및 요청과 관련된 개인 정보 보호 규정과 같은 요청에 대한 방향을 지정하는 기본 정보를 표시합니다.

##### <a name="progress"></a>진행률

**진행률** 카드는 프로세스의 각 단계를 나열합니다. 데이터 예측, 데이터 검색, 데이터 검토, 보고서 생성 및 요청 닫기. 단계 옆에 채워진 파란색 원은 현재 진행 중인 단계를 나타냅니다. 파란색 원 안의 확인 표시는 단계가 완료되었음을 의미하며 채워지지 않은 원은 단계가 아직 시작되지 않았음을 의미합니다.

##### <a name="total-number-of-items-found"></a>찾은 총 항목 수

현재 진행 단계에 대한 통계입니다. 이 타일은 데이터 예측 요약, 검색에서 발견된 항목 수 및 Microsoft 365 위치 또는 내보내기 상태와 같은 정보를 표시할 수 있습니다.

##### <a name="priority-items-to-review"></a>검토할 우선 순위 항목

**검토할 우선 순위 항목은 검토를** 시작할 때 우선 순위를 지정할 수 있는 항목을 표시합니다. 타일에는 다음 범주에 속하는 항목 수가 표시됩니다.
- **기밀**: Microsoft [민감도 레이블](/microsoft-365/compliance/sensitivity-labels) 이 적용된 항목입니다. 예를 들어 "매우 기밀" 레이블이 있는 Word 문서입니다. 
- **다중 사용자 데이터**: 이러한 항목에는 두 명 이상의 개인 데이터가 포함되어 있습니다. 이러한 항목을 최종 데이터 패키지의 일부로 포함하려면 파일에서 관련 없는 데이터를 수정해야 합니다. 자세한 내용은 [3단계: 아래 데이터를 검토](#step-3-review-data) 하세요. Priva가 다중 사용자 데이터가 있는 항목을 식별하려면 조직에서 [주체 권한 요청에 대한 데이터 일치를](subject-rights-requests-data-match.md) 설정해야 합니다.

**우선 순위 항목을 찾는 방법:**

먼저 아래 단계에 따라 **데이터 수집** 항목 테이블에서 해당 항목의 보기를 사용하도록 설정했는지 확인합니다.

- **수집된 데이터** 탭의 항목 목록 맨 위에 있는 **열 사용자 지정** 을 선택합니다.
- **열 편집** 플라이아웃 창에서 **우선 순위 유형** 옆에 검사를 배치합니다.
- **적용** 을 선택합니다. 이제 항목 목록에 **우선 순위 형식 열이** 있습니다.

이제 우선 순위 항목을 식별하고 우선 **순위 형식** 열을 정렬하여 유사한 형식을 그룹화하여 찾을 수 있습니다.

### <a name="understand-progress-stages"></a>진행 단계 이해

주체 권한 요청은 여러 단계를 거행합니다. 일부 상태는 자동으로 진행되며, 주체 권한 요청 관리자 및 기여자가 파일 검토와 같은 필수 단계를 완료하면 다른 단계가 진행됩니다.

시간이 지남에 따라 또는 여러 기여자가 요청을 수행해야 할 수 있으므로 Priva는 수행할 다음 단계에 대한 상태 및 지침에 대한 지속적인 업데이트를 제공합니다. 이러한 업데이트는 주체 권한 요청 세부 정보 페이지의 **개요** 탭에서 볼 수 있습니다.

#### <a name="data-estimate"></a>데이터 예상
요청을 만들면 Priva는 Microsoft 365 환경에서 데이터 주체와 일치하는 잠재적인 항목을 즉시 찾기 시작합니다. 기준과 일치하는 것으로 생각되는 모든 항목을 식별하면 요청 **개요 페이지의** **데이터 예상 요약** 카드에 추정치가 표시됩니다. 검색 범위 내의 데이터 양은 예상을 완료하는 데 걸리는 시간에 영향을  미칩니다.

요청은 데이터 검색의 다음 단계로 자동으로 이동하며, 모든 콘텐츠 항목이 함께 수집되므로 관련자가 데이터 검토에 대해 공동 작업할 수 있습니다. 경우에 따라 검색으로 이동하기 전에 데이터 예상을 일시 중지하고 계속하기 전에 수행할 다음 단계를 알려 줍니다.

주체 권한 요청을 처음 만들 때 데이터 예측 단계에서 자동으로 일시 중지하도록 선택할 수도 있습니다. 만들기 프로세스 중에 **검색 설정** 단계에서 **예측값 가져오기 첫 번째** 옵션을 선택합니다. [검색 설정 단계에](subject-rights-requests-create.md#define-search-settings) 대한 세부 정보를 검토합니다.

#### <a name="pause-in-data-estimate-for-large-search-results"></a>대용량 검색 결과에 대한 데이터 예상 시간 일시 중지

Priva는 검토할 많은 수의 항목(10K 이상 항목)을 반환할 것으로 예상되는 데이터 추정치를 확인합니다. 결과를 미리 볼 수 있도록 예측이 일시 중지되고 [검색 쿼리를 편집](subject-rights-requests-create.md#refine-your-search) 하여 보다 구체적인 위치 또는 조건을 대상으로 할지 또는 식별된 항목을 계속 검색할지 결정할 수 있습니다.  검색과 일치하는 항목 수와 데이터 볼륨을 화면에 표시합니다. 화면 맨 위에 있는 메시지 표시줄에 다음 옵션 중 하나 또는 둘 다 있습니다.

- **검색 쿼리 편집** 단추는 요청의 검색 설정으로 직접 이동하여 더 엄격한 매개 변수를 설정하고 새 예상치를 생성합니다.
- 검색 쿼리가 300K 항목을 초과하지 않는 한 **데이터를 검색** 하는 옵션도 표시됩니다. 이렇게 하면 검색을 편집하지 않고 데이터를 계속 수집하도록 선택할 수 있습니다.

#### <a name="retrieve-data"></a>데이터 검색
데이터 검색 단계는 데이터 주체의 개인 데이터를 포함하는 모든 파일, 전자 메일, 채팅, 이미지 및 기타 콘텐츠 항목을 검색하고 Azure Blob Storage 컨테이너에 함께 배치하여 검토할 때입니다. 데이터 검색은 데이터 볼륨에 따라 몇 분 또는 훨씬 더 오래 걸릴 수 있습니다. 이 단계가 완료되면 요청이 검토 **데이터의** 다음 단계로 자동으로 이동합니다.

#### <a name="review-data"></a>데이터 검토
 이 단계에서 기여자는 **데이터 수집** 탭에서 결과를 검토하고 편집, 태그 적용 및 노트 추가와 같은 적용 가능한 모든 작업을 수행해야 합니다. 검토가 완료되면 검토 **완료** 를 선택합니다.

#### <a name="generate-reports"></a>보고서 생성
이 단계에서 보고서가 생성됩니다. 완료되면 **보고서** 탭에서 찾을 수 있습니다. 최종 검토 및 요청을 수행한 데이터 주체에게 배달하기 위해 완료된 파일을 내보낼 수 있습니다.

#### <a name="close-the-request"></a>요청 닫기
닫힌 요청은 이 주체 권한 요청을 수행하기 위해 모든 작업이 완료되었음을 나타냅니다. 수집된 모든 데이터와 보고서는 [데이터 보존 설정](priva-settings.md#data-retention-periods)에 따라 보존됩니다.

## <a name="step-2-optional-view-and-edit-search-queries"></a>2단계(선택 사항): 검색 쿼리 보기 및 편집

주체 권한 요청 뒤에 있는 데이터 검색에 대한 자세한 정보를 보려면 **검색 쿼리 세부 정보 보기를** 선택합니다. 그러면 쿼리를 요약하고 발견된 내용에 대한 추가 세부 정보를 표시하는 창이 열립니다.

이 쿼리에 대해 반환될 콘텐츠 유형을 확인하기 위해 **검색 결과를 미리 볼** 수 있는 옵션이 있습니다. 이 검색의 속성을 변경하고 데이터 검색 단계를 시작하지 않은 경우 **검색 쿼리 편집** 옵션을 사용할 수 있습니다.

검색 쿼리 단계별 편집 프로세스를 사용하면 데이터 주체 식별, 검색 필터 및 조건 및 데이터를 찾을 위치(Exchange, SharePoint, OneDrive 및/또는 Teams 포함)에 대한 속성을 변경하거나 추가할 수 있습니다. 이러한 옵션을 사용하여 원하는 수준의 특이성에 도달합니다. **저장** 을 누르기 전에 새 쿼리의 최종 버전을 검토할 수 있습니다.

검색 쿼리 편집을 마치면 이전 검색 결과를 바꾸기 위해 새 검색이 실행됩니다. 그러면 **진행률** 섹션의 상태가 첫 번째 단계인 **데이터 예측** 으로 다시 설정됩니다. 새 검색을 완료하는 데 최대 60분이 걸릴 수 있습니다. 완료되면 요청의 세부 정보 페이지에 업데이트된 결과가 표시됩니다.

## <a name="step-3-review-data"></a>3단계: 데이터 검토

이 단계에서 기여자는 **데이터 수집** 탭에서 결과를 검토해야 합니다. 모든 이해 관계자의 콘텐츠 검토를 용이하게 하기 위해 Teams 채널이 자동으로 설정됩니다. 자세한 내용은 [데이터 검토에 대한 공동 작업을](#collaborate-on-data-review) 참조하세요. 데이터 검토 단계의 필수 작업은 아래에 설명되어 있습니다.

#### <a name="mark-items-as-include-or-exclude-and-add-notes"></a>항목을 포함 또는 제외로 표시하고 노트 추가

검색에서 반환된 식별된 항목 목록을 검토합니다. 항목이 최종 보고서의 일부로 데이터 주체로 다시 포함되도록 결정한 경우 항목 목록의 맨 위에 있는 명령 모음에서 **[포함** ]을 선택합니다. 항목 목록 오른쪽의 콘텐츠 검토 영역에서 파란색 **포함** 단추를 선택할 수도 있습니다. **[포함**]을 선택하면 슬라이드 노트를 추가하는 옵션이 있는 플라이아웃 창이 나타납니다. 완료되면 **제출** 을 선택하여 항목의 검토 상태를 **포함** 으로 저장합니다.

항목이 요청의 일부로 속하지 않는 경우 명령 모음에서 **제외** 를 선택하거나 콘텐츠 검토 영역의 **제외** 단추를 선택합니다. 항목을 제외하면 [데이터 주체에 대해 생성된 최종 보고서에](subject-rights-requests-reports.md) 포함되지 않습니다.

> [!NOTE]
> 항목을 **제외** 로 표시하는 경우 주체 권한 요청과 관련이 없는 이유에 대한 근거로 메모를 추가해야 합니다. 참고는 내부용이며 최종 보고서에 포함되지 않습니다.

콘텐츠가 가양성으로 표시되는 경우 **일치 안** 을 선택하고 플라이아웃 창에서 **확인을** 선택합니다. 이 작업은 최종 보고서에서 파일을 제외하고 검색에서 검색되지 않아야 하는 항목으로 플래그를 지정합니다.

#### <a name="apply-tags"></a>Apply tags

태그를 사용하여 추가 주의가 필요한 항목을 식별할 수 있습니다. Priva는 설명을 설정할 수 있는 **후속** 작업, **삭제** 및 **업데이트** 라는 세 가지 기본 태그를 제공합니다. Priva는 이름을 지정하고 설명할 수 있는 두 개의 사용자 지정 태그도 제공합니다.

예를 들어 데이터 검토 중에 조직에서 콘텐츠 항목을 유지할 필요가 없다고 판단하는 경우 **삭제** 태그를 적용한 다음 태그가 지정된 모든 파일 목록을 내보내서 요청이 완료되면 다시 돌아가서 식별된 항목을 삭제할 수 있습니다.

**설정** 관리하는 5개의 태그는 모든 주체 권한 요청에 적용됩니다.

**태그를 추가하거나 제거하려면 다음을 수행합니다.**

- 요청의 **데이터 수집** 탭에 있는 목록에서 항목을 선택합니다.
- 목록 오른쪽의 항목 미리 보기 영역에서 아래쪽 행에서 **태그 적용** 단추를 선택합니다. 항목 이름 오른쪽에 있는 세 개의 점을 선택하고 **태그 적용** 옵션을 선택할 수도 있습니다.
- 태그 목록과 함께 플라이아웃 창이 나타납니다. 항목에 적용할 태그 옆에 있는 확인란을 선택합니다. 확인란을 선택 취소하면 태그가 제거됩니다.
- 완료되면 **저장** 을 선택하여 태그 선택을 저장하고 플라이아웃 창을 닫습니다.

**사용자 지정 태그를 추가하거나 태그 설명을 업데이트하려면 다음을 수행합니다.**
- 주체 권한 요청 페이지에서 화면의 오른쪽 위 모서리에 있는 **설정** 선택하여 Priva 설정으로 이동합니다.
- **데이터 검토 태그** 페이지로 이동하고 태그를 선택하여 설명을 입력하고 사용자 지정 태그의 이름을 입력합니다. [태그 설정](priva-settings.md#data-review-tags)에 대해 자세히 알아봅니다.

**태그가 지정된 항목 목록을 내보내려면 다음을 수행합니다.**
- 주체 권한 요청에서 **수집된 데이터** 페이지로 이동합니다.
- 항목 목록 위에서 마우스로 가리킬 때 **내보내** 기라는 아래쪽 화살표 아이콘을 선택합니다.
- Excel 파일은 요청을 검색하여 수집된 모든 항목의 속성을 보여 주는 다운로드됩니다. **태그** 열을 찾아 항목을 식별하고 태그별로 정렬합니다.

#### <a name="use-the-annotate-command-to-redact-text"></a>주석 달기 명령을 사용하여 텍스트 수정
콘텐츠 검토 영역의 **주석 달기** 명령을 사용하면 인라인 태그를 만들고 콘텐츠 항목 내에서 데이터를 수정할 수 있습니다. 예를 들어 다른 데이터 주체의 개인 정보도 포함된 개인용 파일을 포함해야 하는 경우 명령 모음의 그리기 단추 아래에서 **영역 편집** 을 사용하여 요청을 한 사람과 관련되지 않은 모든 정보를 차단할 수 있습니다. 편집이 완료되면 **[포함** ]을 선택하여 수정된 파일을 요청에 추가합니다. 주석은 Azure Blob에 저장된 파일의 복사본을 만듭니다. 원래 파일은 변경되지 않고 원래 위치에 저장됩니다.

#### <a name="enter-notes-about-a-file"></a>파일에 대한 노트 입력
항목에 메모를 추가하거나 검토하려면 해당 행에서 항목을 선택하고 오른쪽의 콘텐츠 검토 영역에 있는 **파일 노트** 탭으로 이동합니다. **파일 메모 추가** 옵션을 사용하여 새 메모를 만들 수도 있습니다. 전체 사례 수준에서 노트를 검토하거나 추가하려면 위의 주 **메모** 탭으로 이동하여 **사례 메모 추가** 를 사용합니다. 이러한 메모는 요청 작업을 하는 사용자에게 표시되지만 최종 보고서에 포함되거나 데이터 주체와 공유되지 않습니다.

#### <a name="complete-the-review"></a>검토 완료

모든 항목을 검토하고 해당 상태를 **포함**, **제외** 또는 **일치하지 않음** 으로 설정한 경우 요청의 오른쪽 위 모서리에 있는  **검토 완료** 단추를 선택하여 검토 단계를 종료해야 합니다. 플라이아웃 창에는 데이터의 요약이 표시되고 관련 메모가 추가됩니다. 이러한 노트는 내부 레코드 유지를 위한 것이며 데이터 주체와 공유되지 않습니다.

플라이아웃 창에서 **검토 완료** 를 선택하여 검토 단계를 완료합니다. 의사 결정의 요약은 나중에 **보고서** 탭 아래에 제공됩니다.

### <a name="collaborate-on-data-review"></a>데이터 검토에 대한 공동 작업

Priva는 Microsoft Teams 통해 공동 작업을 지원하여 그룹이 주체 권한 요청에 대해 함께 작업할 수 있도록 합니다. 새 요청을 만들면 Teams 채널이 자동으로 만들어지고 기본적으로 요청에 연결됩니다. 여기서 요청을 논의하고 입력 및 기여를 안전하게 공유할 수 있습니다. 대화에 참가하려면 요청을 열고 **공동 작업자와 채팅** 옵션을 사용합니다. 그러면 Microsoft Teams 열리고 주체 권한 요청의 팀 사이트에 대한 일반 채널 내에 배치됩니다.

팀 사이트를 보고 참가할 수 있는 활성 협력자 목록을 검토하려면 주체 권한 요청 내에서 **협력자** 탭을 엽니다. 이 요청에 대해 공동 작업할 사용자를 추가하려면 **공동 작업자 추가** 옵션을 선택합니다.

주체 권한 요청을 만들 때 Teams 사이트를 생성하는 기본 동작을 변경하려면 위쪽 탐색의 **설정** 이동하여 **Teams 협업** 을 선택하여 설정을 수정합니다.

주체 권한 요청 내에서 오른쪽 위에 있는 **공유** 옵션을 사용하여 Teams 또는 전자 메일을 통해 사용자를 반복하거나 Priva의 페이지에 대한 링크를 복사할 수도 있습니다. Teams 통해 공유하면 계정에서 사용할 수 있는 기존 Teams 사이트 및 채널을 선택할 수 있으며, 여기서 제공하는 메시지와 함께 이 사례에 대한 링크를 게시할 수 있습니다.

## <a name="step-4-close-the-request"></a>4단계: 요청 닫기

주체 권한 요청을 해결하는 데 필요한 모든 작업을 수행한 경우 **요청 닫기를** 선택합니다. 그러면 보고서 탭에서 찾을 수 있는 최종 **보고서가** 만들어집니다. 완료는 요청의 파일 수에 따라 다소 시간이 걸릴 수 있습니다.
