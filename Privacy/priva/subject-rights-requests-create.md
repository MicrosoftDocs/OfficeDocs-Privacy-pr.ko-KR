# <a name="create-a-request-and-define-search-settings"></a>요청 만들기 및 검색 설정 정의

**이 문서에서는** Priva 주체 권한 요청을 수행하기 위한 요청을 만드는 방법을 알아봅니다. 데이터 검색에 대한 검색 설정 및 검색을 구체화하는 방법을 이해합니다.

보기 전용 역할([Priva 권한](priva-permissions.md) 참조)이 아닌 주체 권한 요청 관리자 역할 그룹에서 역할을 보유하는 사용자는 요청을 만들 수 있습니다. 요청을 만드는 방법에는 두 가지가 있습니다.

- [템플릿](#quick-setup-use-a-template-with-default-settings)에서 맞춤형 기본 설정을 사용하는 빠른 "기본 제공" 옵션입니다. 또는
- 모든 설정을 안내하는 단계별 프로세스인 [사용자 지정](#custom-setup-guided-process-to-choose-all-settings) 옵션입니다.

> [!TIP]
> **주체 권한 요청** 솔루션을 처음 방문할 때 요청을 만들고 기능을 탐색하는 간소화된 방법을 제공합니다. 아래 [의 첫 번째 요청 시작을 참조하세요](#getting-started-with-your-first-request) .

## <a name="request-types"></a>요청 유형

Priva 주체 권한 요청 세 가지 유형의 요청을 지원합니다.

1. **액세스**: Microsoft 365 조직에서 보유한 데이터 주체의 개인 정보에 대한 요약을 제공합니다.

2. **내보내기**: 데이터 주체의 개인 정보가 포함된 콘텐츠 항목의 요약 및 내보낸 파일을 제공합니다. 검색 설정에서 수집한 데이터를 검토하는 동안 검토되고 **포함** 됨으로 표시된 항목입니다.

3. **추가 작업을 위해 태그가 지정된 목록**: 데이터 검토 중에 태그가 지정된 파일의 요약을 생성하며 Priva 외부에서 추가 작업이 필요할 수 있습니다. 예를 들어 요청에 따라 데이터 주체의 개인 정보를 쉽게 삭제해야 할 수 있습니다. 포함된 태그를 보고 [Priva 설정](priva-settings.md)에서 조직에 대한 사용자 지정 태그를 설정할 수 있습니다.

## <a name="getting-started-with-your-first-request"></a>첫 번째 요청 시작

주체 권한 요청의 평가판 또는 구독을 시작하면 기본 설정을 사용하는 첫 번째 요청에 대한 간단한 기본 설정이 제공됩니다. 이 설정은 주체 권한 요청 워크플로를 탐색하고 해당 기능을 숙지하는 데 도움이 될 수 있습니다.

주체 권한 요청 페이지에 처음 도착하면 맨 위에 **시작** 단추가 있는 배너가 표시됩니다. 사용자가 이 단추를 선택하면 해당 사용자의 정보가 이름 및 전자 메일 필드에 미리 채워진 플라이아웃 창이 나타나고 모든 기본 설정이 표시됩니다.

**정보를 사용하여 요청 기능 탐색**: 사용자 고유의 정보를 기반으로 주체 권한 요청을 시도하면 프로세스의 각 단계를 진행하는 데 익숙해지고 편안함을 얻을 수 있습니다. 기본 검색이 반환되는 내용을 확인하고 검색 설정을 조정하여 결과를 구체화하는 방법을 연습할 수 있습니다. **데이터 수집** 탭에서 오른쪽에 있는 미리 보기 영역의 항목을 검토하고 텍스트 수정, 태그 적용, 노트 입력, 최종 보고서에 포함하거나 제외할 항목 표시([주체 권한 요청에 대한 검토 데이터](subject-rights-requests-data-review.md)에서 세부 정보 찾기)를 연습할 수 있습니다.

- 정보를 사용하여 첫 번째 요청을 만들 필요가 없습니다. 데이터 주체에 대한 요청을 시작할 준비가 되면 이름 및 전자 메일 주소를 데이터 주체의 정보로 바꿉니다.

모든 설정을 수락하고 요청을 만들려면 **만들기** 를 선택합니다. 창이 닫히면 **주체 권한** 요청 페이지에 새 요청이 나열됩니다. 요청을 만들기 전에 기본 설정을 변경하려면 **요청 세부 정보 편집** 을 선택하여 [주체 권한 요청 만들기 마법사](#custom-setup-guided-process-to-choose-all-settings)로 전환합니다.

> [!NOTE]
> 만든 모든 요청은 요청에 사용되는 데이터 주체의 정보에 관계없이 평가판 또는 유료 구독 할당에 포함됩니다. 표준 30일 데이터 보존 기간은 요청이 닫힌 후에 적용됩니다. [주체 권한 요청에 대한 보존 기간을](subject-rights-requests-reports.md#retention-periods-for-reports-and-data) 변경하는 방법을 알아봅니다.

## <a name="quick-setup-use-a-template-with-default-settings"></a>빠른 설정: 기본 설정이 있는 템플릿 사용

템플릿에서 직접 요청을 만들 때 기본 설정은 신속하게 시작하고 실행할 수 있도록 하기 위한 것입니다. 세 가지 템플릿은 데이터 **액세스**, **데이터 내보내기** 및 **추가 작업을 위해 태그가 지정된 데이터** 라는 [세 가지 요청 형식](#request-types)에 해당합니다.  템플릿의 기본 설정을 보고 요청 생성 프로세스 중에 변경할 수 있습니다.

##### <a name="selecting-the-relationship"></a>관계 선택

각 템플릿을 사용하면 데이터 주체와 조직 간의 관계 유형을 선택할 수 있습니다. 그러면 기본 설정이 결정됩니다. 아래 포인트는 선택한 관계가 템플릿의 검색 설정에 미치는 영향을 설명합니다.

- **현재 직원**: 검색 결과는 직원이 참여한 전자 메일 또는 Teams 채팅 또는 직원이 작성한 파일을 방지하기 위해 조정됩니다. 이 설정은 현재 직원이 일반적으로 작성한 콘텐츠 또는 참여한 통신에 액세스할 수 있으므로 검색 결과를 간소화합니다.

- **이전 직원**: 검색 설정은 이전 직원의 사서함(사용 가능한 경우)과 직원이 작성한 항목의 우선 순위를 지정합니다. 이러한 설정은 이전 직원이 생성하거나 참여한 콘텐츠를 반환하는 것을 목표로 합니다.

- **고객**, **잠재 직원** 및 **기타**: 이러한 관계에 대한 검색 결과에는 데이터 주체가 작성한 콘텐츠가 포함되지 않으며 최신 버전의 SharePoint 항목만 포함됩니다([고급 검색 옵션](#advanced-search-options)에서 자세히 알아보기).

> [!TIP]
> 주 **주체 권한 요청** 페이지의 **조직 관계** 열에서 관계별로 요청 목록을 정렬할 수 있습니다.

#### <a name="steps-for-creating-a-request-from-a-template"></a>템플릿에서 요청을 만드는 단계

1. [Microsoft Purview 규정 준수 센터의](https://compliance.microsoft.com/) 왼쪽 탐색 영역에서 **Priva 주체 권한 요청** 선택합니다.

2. 화면의 오른쪽 위 모서리에서 **요청 만들기를** 선택합니다.

3. 만들려는 요청 유형(**데이터 액세스**, **데이터 내보내기** 또는 **추가 작업을 위해 태그가 지정된 데이터**)을 찾고 **시작** 선택합니다. 플라이아웃 창이 나타납니다.

4. **조직과의 관계에서** 데이터 주체와 조직 간의 [관계를](#selecting-the-relationship) 설명하는 옵션을 선택합니다. **기타** 를 선택하면 플라이아웃 창에 **다른 관계** 텍스트 필드가 나타납니다. 조직에 적합한 용어를 입력합니다. 예를 들어 "patient" 또는 "student"입니다. 

5. 모든 기본 설정을 검토하고 변경하려면 **설정 보기** 단추를 선택합니다. 설정이 플라이아웃 창에 표시됩니다. 설정을 변경하려면 아래 설명된 [단계별 프로세스](#custom-setup-guided-process-to-choose-all-settings)로 안내하는 **설정 편집** 을 선택합니다.

6. 데이터 주체의 세부 정보를 입력합니다.
    - 이름과 성, 전자 메일 주소는 필수 필드입니다. 현재 및 이전 직원의 경우 데이터 주체 검색 필드에 **데이터** 주체의 이름을 입력하면 선택할 사용자 목록이 표시됩니다. 해당 필드 아래의 링크를 선택하여 이름과 전자 메일을 수동으로 입력할 수도 있습니다.
    - 데이터 주체의 거주 국가, 요청과 관련된 개인 정보 보호 규정 및 기간을 입력하는 것은 선택 사항입니다.
   
    > [!TIP]
    >기본 기간은 지난 12개월 이내에 생성되거나 수정된 콘텐츠를 검색합니다. 언제든지 만든 모든 콘텐츠를 선택하거나 사용자 지정 기간을 설정하도록 이 설정을 변경할 수 있습니다.

7. 완료되면 **만들기** 를 선택합니다. 요청 목록 맨 위에 새 요청이 나열된 **주체 권한 요청** 페이지로 돌아갑니다.

기본적으로 요청의 이름은 데이터 주체의 이름과 요청 유형으로 지정됩니다. 요청 이름을 편집하려면 목록에서 요청을 선택하여 세부 정보 페이지를 열고 화면 맨 위에 있는 **편집** 명령을 선택합니다. 요청 만들기 마법사에 도착합니다. 이름을 편집하고 설명을 추가할 수 있는 **요청 이름** 페이지로 진행할 때까지 **다음** 을 선택합니다.

[데이터 예측 단계에서 템플릿이 자동으로 일시 중지](subject-rights-requests-data-retrieval.md#pause-in-data-estimate)되지 않습니다. 그러나 템플릿의 플라이아웃 창에서 **보기 설정** 선택하여 이 설정 및 기타 [검색 설정을](subject-rights-requests-create.md#defining-search-settings) 변경할 수 있습니다.

## <a name="custom-setup-guided-process-to-choose-all-settings"></a>사용자 지정 설정: 모든 설정을 선택하는 단계별 프로세스

사용자 지정 요청 옵션은 정책을 만들기 위한 단계별 프로세스입니다. 먼저 템플릿을 선택한 다음 각 설정을 안내하여 정책을 사용자 지정합니다. 아래 지침에서는 세 가지 정책 유형 각각에 적용되는 기본 설정에 대한 세부 정보를 제공합니다. 설정이 정책 유형에 따라 다른 경우 특정 지침에 연결합니다.

아래 단계에 따라 요청을 만듭니다.

1. [Microsoft Purview 규정 준수 포털](https://compliance.microsoft.com/) 왼쪽 탐색 영역에서 **Priva 주체 권한 요청** 선택합니다.

2. 화면의 오른쪽 위 모서리에서 **요청 만들기를** 선택합니다.

3. **사용자 지정** 옵션에서 **시작** 선택합니다. 요청 만들기 마법사로 이동됩니다.

4. **데이터 주체 정보** 페이지에서 데이터 주체의 이름과 성 및 전자 메일 주소를 입력합니다. **거주 추가** 를 선택하여 데이터 주체의 거주 국가를 선택합니다. 데이터 주체가 조직과 어떻게 관련되어 있는지를 식별하는 옵션을 선택합니다. **기타** 를 선택하는 경우 조직에 적합한 관계를 설명하는 용어를 입력합니다. 예를 들어 "patient" 또는 "student"입니다. **다음** 을 선택하여 다음 단계로 이동합니다.

5. **위치** 페이지에서 데이터 주체의 정보를 검색할 위치를 결정합니다. 각 옵션 옆에 있는 상태 토글 스위치를 **켜** 기 위치로 이동하여 다음 위치 중 하나 또는 둘 다를 선택합니다.

   - **Exchange**: Exchange 사서함 및 개별 또는 그룹 Teams 채팅에서 데이터를 찾습니다. 조직의 모든 Exchange 계정을 검색하도록 선택하거나 계정 **선택을** 선택하여 **Exchange 사서함** 플라이아웃 창에서 개별 사용자를 선택할 수 있습니다.

   - **SharePoint**: SharePoint 사이트, 비즈니스용 OneDrive 사이트 및 Teams 채널에서 데이터를 찾습니다. 조직의 모든 SharePoint 사이트를 검색하도록 선택하거나 **사이트 선택을** 선택하여 **SharePoint 사이트** 플라이아웃 창에서 개별 사용자를 선택할 수 있습니다.

    > [!TIP]
    > 적절한 검색어 식별에 대한 도움말은 다음 항목을 참조하세요.
    > - SharePoint 사이트 및 URL: [SharePoint 관리 센터의 사이트 관리](/sharepoint/manage-sites-in-new-admin-center)에서는 사이트를 정렬 및 필터링하는 방법과 SharePoint 사이트를 검색하는 방법에 대한 지침을 제공합니다. 이 URL을 찾아 **SharePoint 사이트** 플라이아웃 창의 검색 필드에 입력합니다.
    > - Teams 채팅 및 채널: [Get-Team](/powershell/module/teams/get-team)은 특정 속성 또는 정보를 제공하여 Microsoft Teams 팀을 찾는 방법을 보여줍니다.
    > - OneDrive 사이트 및 URL: [OneDrive 정보 URL](/onedrive/list-onedrive-urls#about-onedrive-urls)은 사용자의 OneDrive URL에 대한 적절한 형식 및 속성에 대한 정보를 제공합니다. 검색에서 OneDrive 사이트를 식별하는 데 도움이 됩니다.

6. **검색 설정 정의** 페이지에서 다양한 고급 검색 옵션 중에서 선택하여 기본 검색을 변경하도록 선택할 수 있습니다. 데이터가 자동으로 반환되기 전에 먼저 예측값을 가져오기로 선택할 수도 있습니다. 이러한 옵션 중 하나를 선택하면 **다음** 을 선택하면 추가 화면으로 진행됩니다. 아래 [검색 설정 정의](#defining-search-settings) 에서 세부 정보를 가져옵니다. 검색을 변경하지 않으려면 모든 옵션을 비워 두고 **다음** 을 선택하여 다음 단계로 이동합니다.

7. **요청 유형 선택 페이지에서 후속** 작업을 위해 **요청** 유형(액세스, **내보내기** 또는 **태그가 지정된 목록**)을 선택합니다([위의 설명 참조](#request-types)). 요청이 데이터 개인 정보 보호 규정과 관련된지 여부를 나타냅니다. 완료 기한을 검토하거나 변경합니다. 이는 기본적으로 요청 생성 날짜 2주 이전입니다. 그런 후 **다음** 을 선택합니다.

8. **이 요청 페이지의 이름 확인 또는 편집에서 요청에** 대해 제공된 친숙한 이름을 유지하거나 편집하고 선택적 설명을 입력할 수 있습니다. 그런 후 **다음** 을 선택합니다.

9. **검토 및 완료** 페이지에서 이전 단계에서 입력한 내용의 요약을 검토합니다. 각 섹션에서 **편집 링크를** 선택하여 모든 필드를 편집할 수 있습니다. 완료되면 **요청 만들기** 를 선택합니다.

요청이 만들어지면 확인 화면이 표시됩니다. **완료** 를 선택하여 주 주체 권한 요청 화면으로 돌아갑니다. 새 요청은 요청 목록의 맨 위에 나열됩니다. 목록에서 선택하여 진행 단계 검토 및 작업을 시작합니다.

#### <a name="what-happens-after-your-request-is-created"></a>요청을 만든 후 수행되는 동작

검색 매개 변수를 기반으로 찾게 될 예상 데이터 양을 즉시 계산합니다. 예상 결과 및 요청이 다음 단계로 이동했는지 여부를 확인하려면 몇 분에서 1시간 정도의 요청 세부 정보 페이지를 방문하세요. 요청은 다음과 같은 경우를 제외하고 **데이터 예상** 데이터 검색에서 **데이터 검색** 으로 자동으로 진행됩니다.

 - 5단계, **검색 설정 정의** 에서 먼저 데이터 esimtae를 수신하도록 선택한 경우 모든 데이터를 검색하기 전에 검색에 대한 세부 정보를 보고 검색을 변경할 수 있습니다. **데이터 예상** 시간에서 일시 중지한 경우 **데이터 검색** 의 다음 단계로 수동으로 진행해야 합니다.
 
 - 데이터 예측에서 중지하도록 선택하지 않았지만 검색 시 대량의 데이터가 생성될 것으로 예상하는 경우 데이터 **검색** 을 계속하기 전에 검색을 편집할 수 있는 링크가 있는 메시지 표시줄이 요청 맨 위에 표시됩니다.

[데이터 예측 및 검색](subject-rights-requests-data-retrieval.md)을 방문하여 이러한 단계에 대해 자세히 알아보세요.

## <a name="defining-search-settings"></a>검색 설정 정의

주체 권한 요청을 만들면 만들기 마법사의 **위치** 페이지에서 선택한 항목에 따라 기본 검색이 실행됩니다. 더 많은 대상 검색을 수행하거나 콘텐츠 항목을 검색하기 전에 예상 데이터를 가져오도록 선택하려는 경우 요청 만들기 마법사의 **검색 설정** 페이지에서 이러한 항목을 선택할 수 있습니다.

### <a name="advanced-search-options"></a>고급 검색 옵션

- **검색 구체화**: 이 옵션을 사용하면 조직의 데이터 중 데이터 주체를 식별하는 데 도움이 되는 추가 속성을 지정할 수 있습니다. 이 옵션을 선택하면 검색 구체화에서 아래에 설명된 더 많은 검색 매개 변수를 추가하라는 메시지가 [표시됩니다.](#refining-your-search)
- **데이터 주체가 작성한 콘텐츠 포함**: 이 옵션은 데이터 주체가 작성한 콘텐츠를 찾습니다. 예를 들어 데이터 주체에 의해 생성되거나 SharePoint 업로드된 파일이 있습니다. 이 옵션을 선택하면 반환되는 데이터의 양이 크게 증가할 수 있습니다.
- **모든 버전의 항목을 포함합니다**. SharePoint 검색 위치로 선택한 경우 기본 검색 쿼리는 현재 버전의 SharePoint 항목만 반환합니다. 이 확인란을 선택하면 현재 및 모든 이전 버전의 SharePoint 항목이 반환되므로 검토할 데이터가 훨씬 더 많이 생성됩니다.

> [!TIP]
> Priva 정확한 데이터 값을 기반으로 데이터 주체를 식별할 수 있도록 추가 자료를 업로드하도록 선택할 수 있습니다. 자세한 내용은 [주체 권한 요청에 대한 데이터 일치를 참조하세요](subject-rights-requests-data-match.md).

### <a name="data-estimate"></a>데이터 예상

**예측 가져오기 첫 번째** 옵션은 데이터가 자동으로 검색되기 전에 얼마나 많은 데이터를 찾을 것으로 예상되는지 예측합니다. 예상 비용이 요청의 세부 정보 페이지에 표시되면 검색 결과를 보고 검색된 항목의 샘플링을 미리 볼 수 있습니다. 항목이 예상 결과를 나타내는 경우 콘텐츠 항목의 실제 검색을 진행하려면 **데이터 검색** 을 선택해야 합니다. [데이터 예측 단계에](subject-rights-requests-data-retrieval.md) 대한 자세한 내용을 확인합니다.

## <a name="refining-your-search"></a>검색 구체화

**검색 설정** 페이지에서 **검색 구체화** 를 선택하거나 **데이터 예측** 단계에서 일시 중지하고 검색 쿼리를 편집하도록 선택한 경우 검색 결과를 추가로 대상으로 지정하기 위해 개인 특성 및 조건에 대한 세부 정보를 제공하라는 메시지가 표시됩니다. 두 범주 중 하나 또는 둘 다에서 추가할 수 있습니다. 이 섹션에서 선택을 완료하면 요청에 대한 데이터 검색은 검색 설정을 기반으로 합니다.

아래에 설명된 세부 정보는 **데이터 예측** 단계에서 일시 중지하고 검색 쿼리를 편집하도록 선택한 경우 제공하라는 메시지가 표시됩니다. 

#### <a name="add-personal-attributes"></a>개인 특성 추가

데이터 주체 이름, 애칭, 전자 메일 주소 및 주소의 텍스트 필드에 값을 입력합니다. 생년월일 또는 전화 번호와 같은 다른 식별자를 추가할 수 있으며 텍스트 필드는 세미콜론으로 구분된 여러 항목을 지원합니다. 완료되면 **다음** 을 선택하여 **조건** 페이지로 계속 진행합니다.

#### <a name="conditions"></a>조건

**조건 추가** 단추를 선택하여 항목 이름, 보낸 사람 및 받는 사람 이름, 개인 데이터 형식 및 항목이 조직 외부에서 외부적으로 공유되었는지 여부를 포함하여 검색 대상을 추가로 지정할 조건 범위 중에서 선택합니다. 텍스트 필드는 세미콜론으로 구분된 여러 항목을 지원합니다. 완료되면 **다음** 을 선택하여 검색 설정 및 진행률을 요청 유형 설정으로 저장합니다.
